Qt Palmtop FAQ

  1. How do I open files? (MP3s, MPEGs, etc.)

    Currenltly, you can use the File Browser to find you file and click-and-hold to bring up a menu where you can then Add to Documents, and the file will appear in the Documents tab.

    The File Browser can Open files directly (ie. invoke applications to edit them).

    Qt Palmtop supports a document model which is not directly keyed to files. Of course, ultimately files need to be accessed, but the interface to that is still under development.

    The advantages of having a layer above the file system include:

    • Two documents can have the same name (eg. you could have lots of "Letter to Dad" documents, and you might access them by date or by a search engine, rather than by ever-more-ridiculously complex "Letter to Dad about dog", or the more typical "letter1", "letter2", "letter3").
    • Categorization doesn't have to be partitioning (eg. you "Letter to Dad" documents might be categorized in the "Dad" category and the "Letter" category, while in a traditional hierarchical filesystem you would have to decide on ONE structure from the possible choices of "letters/dad" and "dad/letters"; if you chose the first option, then your letters to mum would be conveniently in "letters/mum", but pictures of Dad would be in a completely different area - "pictures/dad"; the reverse problem arises with the second choice.
