Detailed Description

The QCopEnvelope class encapsulates QCop message sending.

QCop messages allow applications to communicate with each other. These messages are send using QCopEnvelope, and received by connecting to a QCopChannel.

To send a message, use the following protocol: 

     QCopEnvelope e(channelname, messagename);
     e << parameter1 << parameter2 << ...;

For messages without parameters, you can simply use: 

     QCopEnvelope (channelname, messagename);

Do not try to simplify further as some compilers may not do as you expect.

The channelname of channels within Qt Palmtop all start with "QPE/". The messagename is a function identifier followed by a list of types in parentheses. There are no spaces in the message name.

To receive a message, you will generally just use your applications predefined QPE/Application/appname channel (see QPEApplication::appMessage()), but you can make another channel and connect it to a slot with: 

      myChannel = new QCopChannel( "QPE/FooBar", this );
      connect( myChannel, SIGNAL(received(const QCString &, const QByteArray &)),
               this, SLOT(fooBarMessage( const QCString &, const QByteArray &)) );

See also, the list of Qt Palmtop messages.
